```
[ScoutAgent] → [AnalystAgent] → [StrategistAgent] → [FixerAgent]
     ↓               ↓                  ↓                  ↓
 Stealth Nmap    CVE Database     3-Step Reasoning    Bash + Ansible
```

```bash

# Verification
```

## SLIDE 7 — Key Design Decisions

**Headline:** Engineering Choices That Matter

**Three callout cards:**

**1. Deterministic Reporting**
> Tables, CVE findings, and port data are built from typed Python state — not LLM output. The LLM only writes narrative prose. This eliminates hallucinated CVEs and fabricated risk ratings.

**2. Safety-First Execution**
> The Safety Filter inspects every generated script for reboot commands, critical service restarts, and firewall flushes before they reach the operator. The Approval Gate blocks all execution until a human confirms.

**3. Graceful Degradation**
> Scapy unavailable → falls back to nmap ping sweep. LLM JSON parse fails → deterministic template-based fallback. Every component has a tested fallback path.
